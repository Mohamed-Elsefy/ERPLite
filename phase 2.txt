ERPLite — Phase 2 Summary
Repository Layer + Data Access Architecture

هذه المرحلة تمثل بناء طبقة الوصول للبيانات (Data Access Layer) بالكامل داخل مشروع ERPLite باستخدام:

Repository Pattern
Generic Repository
Unit Of Work
Query Extensions
Soft Delete Architecture
Transaction Management
Dependency Injection

الهدف الأساسي من هذه المرحلة هو:

فصل الوصول للبيانات عن الـ Business Logic
منع استخدام DbContext مباشرة خارج الـ Repository Layer
بناء بنية احترافية قابلة للتوسع
تجهيز المشروع للـ Services Layer والـ Authentication والـ Financial Operations

أولًا — Generic Repository Architecture
IGenericRepository<T, TKey>

تم إنشاء:

IGenericRepository<T, TKey>

بدل:

IGenericRepository<T>

لدعم:

int Keys
string Keys الخاصة بـ ASP.NET Identity
السبب

لأن:

ApplicationUser : IdentityUser<string>

يستخدم:

string Id

ولذلك تم اعتماد Generic Key Architecture.

العمليات الموجودة داخل IGenericRepository
Read Operations

Task<IEnumerable<T>> GetAllAsync();

    Expression<Func<T, bool>> predicate);

    Expression<Func<T, bool>> predicate);

    Expression<Func<T, bool>> predicate);

    Expression<Func<T, bool>>? predicate = null);
